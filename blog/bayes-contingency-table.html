1. Uniform Prior on (p1,p2)(p_1, p_2)(p1​,p2​)

One of the more dubious statistical tests I've come across is Fisher's exact test. So I did a quick Google search, and apparently Andrew Gelman agrees. I don't think his suggestion i) (model the 2x2 contingency table with two independent binomial distributions) is generally applicable though. This is because in contingency table applications, I'd want more weight on the "null hypothesis" p1=p2p_1 = p_2p1​=p2​, which would imply non-independence. Indeed, the uniform distribution over [0,1]×[0,1][0,1] \times [0,1][0,1]×[0,1] only places 50% likelihood on a risk ratio between 1/2 and 2, and 75% likelihood on an absolute difference of less than .5:

Accordingly, Bob Carpenter's blog post using this prior is quite generous, placing over 90% likelihood on men having a higher percentage of left-handedness than women given the following toy data from Wikipedia:

2. Bayesian Logistic Regression

We can actually encode the conservative prior belief that handedness is unlikely to significantly differ between males and females, by making the logit model Bayesian. We place a weakly informative N(0,1)\mathcal{N}(0,1)N(0,1) prior on the coefficient, which is the log-odds ratio, and simulate the uniform marginals that we had before by using a Logistic(0,1)\text{Logistic}(0,1)Logistic(0,1) prior on the intercept (in most settings, I'd pick tighter priors, but for this toy data I'm interested in the impact of just the coefficient prior). The marginal and joint priors for (p1,p2)(p_1, p_2)(p1​,p2​) look like the following (using seaborn.kdeplot) - note that the marginal isn't perfectly flat, since adding in the coefficient fattens up the distribution of logit⁡(p1)\operatorname{logit}(p_1)logit(p1​), but it's close enough:

In Stan, we can code the model as:

data {
  real x1;
  real x2;
  int<lower=0> y1;
  int<lower=0> n1;
  int<lower=0> y2;
  int<lower=0> n2;
}
parameters {
  real intercept;
  real coeff;
}
model {
  intercept ~ logistic(0, 1);
  coeff ~ normal(0, 1);
  y1 ~ binomial_logit(n1, intercept + coeff * x1);
  y2 ~ binomial_logit(n2, intercept + coeff * x2);
}
And we can run the model on our toy data in rstan:
> stan_dat = list(x1 = -0.5, x2 = 0.5, y1 = 4, n1 = 48, y2 = 9, n2 = 52)
> stan_fit <- stan(file = "lreg.stan", data = stan_dat, chains=4, cores=2, iter=10000)
> samples <- extract(stan_fit, pars=c('intercept', 'coeff'))
> summary(samples$intercept)
   Min. 1st Qu.  Median    Mean 3rd Qu.    Max.
-3.5257 -2.1215 -1.9175 -1.9293 -1.7231 -0.9438
> summary(samples$coeff)
   Min. 1st Qu.  Median    Mean 3rd Qu.    Max.
-1.3620  0.2370  0.5812  0.5880  0.9338  2.9494
> mean(samples$coeff < 0)
[1] 0.12855
> men_logits <- samples$intercept + 0.5 * samples$coeff
> women_logits <- samples$intercept - 0.5 * samples$coeff
> men_p <- invlogit(men_logits)
> women_p <- invlogit(women_logits)
> p_diff <- men_p - women_p
> quantile(p_diff, c(0.005,0.025,0.5,0.975,0.995))
       0.5%        2.5%         50%       97.5%       99.5%
-0.08969971 -0.04901461  0.06404981  0.18018982  0.22328685
> odds_ratio <- (men_p / (1 - men_p)) / (women_p / (1 - women_p))
> quantile(odds_ratio, c(0.005,0.025,0.5,0.975,0.995))
     0.5%      2.5%       50%     97.5%     99.5%
0.4839859 0.6534247 1.7881112 5.1106308 7.2953805

Notice the shrinkage, compared to the frequentist version. We can also see that the resulting 12.9% chance of men being less likely to be left-handed is greater than the corresponding one-sided value from the uniform prior inference, the frequentist logit model, and Fisher's exact test. The 95% credible intervals are also more pessimistic: [-0.049, 0.18] for the absolute difference vs [-0.046, 0.218] from the uniform prior inference, and [0.653, 5.111] for the odds ratio vs a confidence interval of [0.583, 10.928] from Fisher's exact test. The tighter the priors, the more pessimistic we will be.

What would you conclude if you saw this data? (Fun fact: there seems to be a fair amount of actual data on handedness on the Internet, and the conclusion seems to be that men are indeed more likely to be left-handed, along with a handful - no pun intended - of other conclusions. I have not reviewed any of the studies, though).

What if we have an m×nm \times nm×n contingency table? Then we can do the same analysis using a multinomial logit model (which is just n−1n - 1n−1 logits), and dummy variables (m−1m - 1m−1 of them, to avoid multicollinearity). If we had a meta-analysis of many such tables, we could place a hyperprior over the coefficient priors (a.k.a. hierarchical Bayes).

Another side note: logistic regression also works on the data from the Wikipedia page on Simpson's paradox - in fact, Simpson's paradox can be interpreted as omitted variable bias. Another way to understand omitted variable bias is the Frisch–Waugh–Lovell (FWL) theorem - when you partial out other covariates, your causal effect changes.

3. Beta Prior over p1/(p1+p2)p_1 / (p_1 + p_2)p1​/(p1​+p2​)? (feat. Pfizer)

So if we look at the study design for the recent Biontech/Pfizer COVID-19 vaccine results - something interesting jumps out: instead of placing a prior on (p1,p2)(p_1, p_2)(p1​,p2​) (where p1p_1p1​ is the probability that a given patient from the treatment group gets COVID-19, and p2p_2p2​ is the probability that a given patient from the control group gets COVID-19), they place a Beta(0.7,1)\text{Beta}(0.7, 1)Beta(0.7,1) prior on θ=p1/(p1+p2)\theta = p_1 / (p_1 + p_2)θ=p1​/(p1​+p2​). But wait! Isn't the model underparameterized now? We had two parameters, and now we have one! Indeed, if y1y_1y1​ out of n1n_1n1​ patients from the treatment group get COVID-19 and y2y_2y2​ out of n2n_2n2​ patients from the control group get COVID-19, then we don't have a well-defined evidence, as

P(y1,y2∣θ)=∫01B⁡(y1;n1,p1)B⁡(y2;n2,p1(1θ−1))P(p1∣θ) dp1(1) P(y_1, y_2 \mid \theta) = \int_0^1 \operatorname{B}(y_1; n_1, p_1)\operatorname{B}(y_2; n_2, p_1(\dfrac{1}{\theta} - 1))P(p_1 \mid \theta) \,dp_1 \tag{1} P(y1​,y2​∣θ)=∫01​B(y1​;n1​,p1​)B(y2​;n2​,p1​(θ1​−1))P(p1​∣θ)dp1​(1)
can take multiple relative values for fixed values of θ\thetaθ, by varying the distribution of p1p_1p1​. In other words, we can't technically do Bayesian inference on the contingency table, since we don't take into account the base rate of infection.

So how does the reconstructed experiment in Kranz's post work? He transforms the problem into a binomial model on the infected patients, assuming n1=n2n_1 = n_2n1​=n2​, i.e.

P(y1,y2∣θ)=B⁡(y1;y1+y2,θ)(2)P(y_1, y_2 \mid \theta) = \operatorname{B}(y_1; y_1 + y_2, \theta) \tag{2}P(y1​,y2​∣θ)=B(y1​;y1​+y2​,θ)(2)
so that we can do a simple conjugate prior model. This is a clever transformation, though note that it deviates from reality in the sense that it conditions on y1+y2y_1 + y_2y1​+y2​, and in the sense that the binomial distribution models draws with replacement (whereas (1)\text{(1)}(1) is the expression for draws without replacement). For example, if we had n1=100n_1 = 100n1​=100 and y1+y2=150y_1 + y_2 = 150y1​+y2​=150, then we would never see y1=125y_1 = 125y1​=125 despite that having a nonzero probability in the binomial distribution (notice that (2)\text{(2)}(2) doesn't include n1n_1n1​ or n2n_2n2​ at all).

Fortunately, in the Pfizer trial we have n=n1,n2≫y1,y2n = n_1, n_2 \gg y_1, y_2n=n1​,n2​≫y1​,y2​ (43538 participants vs 94 cases, to be precise), so we are effectively drawing the infected patients from an infinite population, and the binomial model is probably not too far off. So this could reconcile how the one-parameter model technically isn't well-defined for contingency tables in general, but could still be used to do asymptotically approximate inference in this particular regime.

You can see this behavior for yourself using toy data (like we did earlier) - just create a contingency table, fix y1,y2y_1, y_2y1​,y2​, and vary n=n1=n2n = n_1 = n_2n=n1​=n2​. The single binomial model always yields the same posterior, but a two binomials or logistic model yields varying results, diverging at small nnn and converging at large nnn.

How large does nnn need to be in general for this to work? If we marginalize over y1+y2y_1 + y_2y1​+y2​ and p1p_1p1​:

so if y1+y2=Cny_1 + y_2 = \sqrt{Cn}y1​+y2​=Cn​, then with a relative error of CCC,

This approximation is reminiscent of the law of rare events, which requires n1,n2≫p1,p2n_1, n_2 \gg p_1, p_2n1​,n2​≫p1​,p2​: "There is a rule of thumb stating that the Poisson distribution is a good approximation of the binomial distribution if nnn is at least 20 and ppp is smaller than or equal to 0.05, and an excellent approximation if n≥100n \ge 100n≥100 and np≤10np \le 10np≤10". Note that this condition is slightly different than what we can observe, but we can still apply the law of rare events starting with the observed n1=n2=n≫y1,y2n_1 = n_2 = n \gg y_1, y_2n1​=n2​=n≫y1​,y2​, by noting that the integrand of (1)\text{(1)}(1) is significant only up to some value p=O(y/n)p = O(y/n)p=O(y/n), due to the Chernoff bound (assuming also that we have prior density in this area). So we can pick δ(θ)\delta(\theta)δ(θ) such that:

It's a bit surprising that we didn't find y=O(n)y = \mathcal{O}(n)y=O(n). Is y=O(n)y = \mathcal{O}(\sqrt{n})y=O(n​) tight? Technically it is, since in the extreme case where y2=0y_2 = 0y2​=0:

P(y1=y∣y1+y2=y,p1,p2)=∏i=0y(n−i)p1(n−i)p1+np2 \begin{aligned} P(y_1 = y \mid y_1 + y_2 = y, p_1, p_2) &= \prod_{i=0}^{y} \dfrac{(n-i)p_1}{(n-i)p_1 + np_2} \\ \end{aligned} P(y1​=y∣y1​+y2​=y,p1​,p2​)​=i=0∏y​(n−i)p1​+np2​(n−i)p1​​​
The binomial approximation p1yp_1^yp1y​ is spot on as θ→1\theta \to 1θ→1, and as θ→0\theta \to 0θ→0 the relative error approaches (using Stirling's approximation):
n!(n−y)!ny≈2πn(ne)n2π(n−y)(n−ye)n−yny=e−y(1−yn)y−n−12≈e−ye−y2n+y+y2n=e−(y2−y2)/n \begin{aligned} \dfrac{n!}{(n-y)!n^y} &\approx \dfrac{\sqrt{2\pi n}\left(\dfrac{n}{e}\right)^n}{ \sqrt{2\pi (n-y)}\left(\dfrac{n-y}{e}\right)^{n - y}n^y} \\ &= e^{-y}\left(1 - \dfrac{y}{n}\right)^{y - n - \frac{1}{2}} \\ &\approx e^{-y}e^{-\frac{y^2}{n} + y + \frac{y}{2n}} \\ &= e^{-(y^2 - \frac{y}{2})/n} \end{aligned} (n−y)!nyn!​​≈2π(n−y)​(en−y​)n−yny2πn​(en​)n​=e−y(1−ny​)y−n−21​≈e−ye−ny2​+y+2ny​=e−(y2−2y​)/n​

In other words, the approximation slightly under-updates the posterior. This is not such a bad outcome, since for example, in the Pfizer study if all 94 cases were in the control group, then we would leave around 22% too much mass on the θ→1\theta \to 1θ→1 tail, but that tail would still be tiny (as Bayesians, we're already winging it when it comes to tail mass).

Is this a common approximation in the medical field? I don't know. This was my first time seeing it though, and the math was pretty cool.

That being said, since Bayesian logit models (and any Bayesian model within the reach of MCMC/Variational Inference) are already easy to fit using a computer, there isn't in general any need to use approximation - maybe numerical stability if we need to deal with extremely large counts, or if we want to define an unusually specific prior on θ\thetaθ (e.g. prior elicitation using quantiles or some other method). Just for fun, I'll try my earlier Stan model on some possible contingency tables for the Pfizer trials. This time I'll use fatter-tailed Cauchy⁡(0,1)\operatorname{Cauchy}(0,1)Cauchy(0,1) prior for the coefficient, to reflect the setting. Here is the resulting prior density for vaccine efficacy:
